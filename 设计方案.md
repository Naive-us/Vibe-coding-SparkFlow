# 项目蓝图 v2.0：灵感微光 (SparkFlow) - 全栈持久化表达重构器

## 1. 升级愿景 & 核心系统架构
本项目由单端 Mock 升级为 **Full-Stack（全栈）架构**，核心引入“多模型动态适配”与“历史记忆流”。
* **前端 (Frontend):** React + Tailwind CSS + React Flow + Framer Motion
* **后端 (Backend):** Python (FastAPI) -> 极其轻量、原生支持异步，适合大模型流式传输。
* **数据库 (Database):** SQLite + SQLAlchemy ORM -> 零配置，单文件存储，完美契合轻量独立 App。

## 2. 后端核心架构：模型工厂模式 (Factory Pattern)
为了让前端能自由切换 GPT-4, Claude, Ollama (本地大模型) 等，后端采用工厂模式解耦。

### 模型提供者基类 (Base Provider)
```python
from abc import ABC, abstractmethod

class BaseLLMProvider(ABC):
    @abstractmethod
    async def generate_mindmap(self, prompt: str, model_name: str) -> dict:
        """所有模型必须返回统一的结构化 JSON"""
        pass


工厂类实现 (LLM Factory)
class LLMFactory:
    _providers = {}

    @classmethod
    def register_provider(cls, name: str, provider_cls):
        cls._providers[name] = provider_cls

    @classmethod
    def get_provider(cls, name: str, api_key: str = None, base_url: str = None) -> BaseLLMProvider:
        provider_cls = cls._providers.get(name.lower())
        if not provider_cls:
            raise ValueError(f"Unsupported provider: {name}")
        return provider_cls(api_key=api_key, base_url=base_url)

3. 轻量级数据库设计 (SQLite Schema)
持久化存储用户的每一次灵感和生成的图谱，杜绝一次性消耗。

表 1：sessions (灵感会话表)

id: String (UUID, 主键)

title: String (系统根据灵感自动生成的简短标题，用于历史列表显示)

raw_input: Text (用户输入的原始大白话)

provider: String (使用的模型厂商，如 openai, ollama)

model_name: String (具体模型名，如 gpt-4o, llama3)

created_at: DateTime (创建时间)

表 2：nodes (图谱节点表)

id: String (主键)

session_id: String (外键，关联 sessions.id)

node_type: String (如 custom_context, custom_anchor, custom_value)

title: String (节点标题：如 破译潜台词)

summary: String (核心骨架词：如 工程控制力)

detail: Text (悬浮展开的详细台词)

pos_x: Float / pos_y: Float (节点在画布上的坐标)

表 3：edges (图谱连线表)

id: String (主键)

session_id: String (外键)

source_id: String (源节点 ID)

target_id: String (目标节点 ID)

### 给你的 Vibe Coding 额外小贴士 💡

1. **为什么用 SQLite？**
   对于这种专注于个人成长、沉淀表达的独立应用，SQLite 会在你用 Cursor 部署时为你省去 90% 的数据库环境报错问题。它只是你项目根目录下的一个 `.db` 文件，你可以随时拷贝它进行备份。
2. **为什么加入本地 Ollama 支持？**
   考虑到你可能会有在离线状态（如高铁上、或者没有网络的环境）下整理思路、准备面试的需求。接入 Ollama（比如在本地跑一个 `llama3` 或 `qwen2`），能让这个“温暖的灵感工具”变成一个**完全属于你的、绝对隐私的离线树洞**。
