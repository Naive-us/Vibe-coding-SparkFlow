# SparkFlow · 灵感微光 — 产品亮点与推广策略

> 一句话定位：**把零散的一句话灵感，自动变成可交互的思维导图，并一键生成可分享的灵感动画。**

---

## 一、产品亮点总结

### 1. 核心体验：从「模糊想法」到「结构化洞察」

传统 AI 对话返回大段文字，用户仍需自己整理结构。灵感微光的核心差异在于：

- 用户只需用自然语言写下灵感（甚至只是一句话）
- AI 自动提炼为 **5–9 个语义节点** 的思维导图
- 节点分为三类，自带信息架构：
  - **核心**（custom_anchor）：想法的中心命题
  - **场景**（custom_context）：背景、问题、对象
  - **价值**（custom_value）：方案、行动、机会

这不是「词云式切碎」，而是围绕「为什么、给谁、怎么做、价值是什么、下一步是什么」进行结构化重构。

### 2. 累计灵感：想法会「生长」，而不是被覆盖

很多工具每次输入都会重置上下文。灵感微光支持 **同一会话内多次追加灵感**：

- 每条新输入会融合进已有图谱，保留核心节点并扩展分支
- 新增节点高亮标记，变化一目了然
- 可拖拽的「累计灵感」浮球面板，随时回顾输入历史
- 支持单条灵感的删除与回溯

这非常贴近真实创意思维过程——灵感往往是碎片式、渐进式出现的。

### 3. 双视图输出：思维导图 + 灵感动画

勾选「同时生成灵感动画」后，产品从「思考工具」升级为「表达工具」：

| 视图 | 作用 |
|------|------|
| **思维导图** | 深度探索、编辑理解、内部讨论 |
| **灵感动画** | 对外演示、社交分享、快速 pitch |

动画支持 16:9 / 9:16 比例切换，适配横屏演示与竖屏短视频。内置播放 / 暂停 / 重播控制，开箱即用。

### 4. Skill 风格体系：同一灵感，多种表达人格

内置 4 套动画 Skill，覆盖不同场景与受众：

- **治愈暖光** — 柔和编辑风，适合生活类、情感类、个人品牌
- **极简动效** — 克制灰阶，适合设计评审、简约提案
- **技术蓝图** — 深色架构风，适合技术方案、系统设计
- **Remotion Storyboard** — 分镜叙事风，适合产品故事、路演脚本

Skill 以 Markdown 规范定义，**可扩展、可替换**，为后续「Skill 市场」留出架构空间。

### 5. 模型无关 + 隐私友好

- 支持 **通义千问、DeepSeek、OpenAI、Anthropic、Ollama 本地** 等主流模型
- API Key **仅存本地浏览器**，不上传服务端
- 可选接入本地 MCP HTML 生成器，动画生成完全离线
- SQLite 本地持久化，会话历史「时光记录」随时找回

对注重数据隐私的团队、独立开发者和企业内网场景，这是实打实的差异化。

### 6. 交互与美学：有温度的创作画布

- 基于 React Flow 的可交互节点画布，支持缩放、拖拽、MiniMap
- 暖木 + 燕麦色系 UI，区别于冷峻的「开发者工具」审美
- Framer Motion 动效：侧边栏、浮球、节点展开均有细腻过渡
- 节点点击展开详情，核心节点始终展示完整信息

产品传递的情绪是 **「微光」** —— 不喧宾夺主，帮用户把想法照亮，而不是替用户思考。

### 7. 技术架构的可信度与扩展性

```
用户输入 → LLM 结构化 JSON → 思维导图渲染 →（可选）Skill/MCP → 灵感动画 HTML
```

- 前端：React + TypeScript + Zustand + Tailwind
- 后端：Spring Boot + JPA + SQLite
- 扩展点：LLM Provider 抽象、Skill Loader、MCP Client
- 一键启动脚本（`start.ps1` / `start.bat`），降低体验门槛

架构清晰、模块解耦，适合作为 **AI 创意工具的基础设施** 继续演进。

## 二、市场机会：为什么这个产品「有潜力」

### 痛点矩阵

| 现有方案 | 问题 | 灵感微光的解法 |
|----------|------|----------------|
| ChatGPT / Claude 对话 | 输出是文字墙，结构靠人脑 | 自动结构化 + 可视化 |
| XMind / MindMeister | 手动建图，启动成本高 | 一句话自动生成 |
| Notion / 飞书文档 | 静态，不适合快速 pitch | 动态灵感动画一键导出 |
| Canva / PPT | 设计门槛高、耗时长 | AI 生成 + Skill 模板 |
| Gamma / Beautiful.ai | 偏演示文稿，弱于思维发散 | 思维导图 + 动画双模式 |

### 目标用户画像

1. **独立开发者 / 创客** — 把产品想法快速可视化，用于社群讨论或众筹 pitch
2. **产品经理** — 需求 brainstorm、竞品分析、方案对比的结构化整理
3. **内容创作者** — 选题策划、脚本大纲、短视频分镜灵感
4. **设计师 / 咨询师** — 客户 workshop 中的实时灵感捕捉与呈现
5. **学生 / 研究者** — 论文选题、课题框架的快速搭建

### 潜在商业模式

- **免费版**：本地部署 + 自带 API Key，社区增长
- **Pro 版**：云端同步、团队协作、更多 Skill、导出 PDF/视频
- **Skill 市场**：设计师/开发者售卖自定义动画风格
- **企业版**：私有化部署、SSO、审计日志、内网 LLM 对接

## 三、产品经理视角：如何推广这个产品

### 阶段一：制造「哇时刻」（0 → 1 认知）

**核心策略：用 15 秒 Demo 说话，而不是用功能列表说话。**

#### 1. 打造 3 个「杀手级 Demo 场景」

| Demo 名称 | 输入示例 | 输出亮点 |
|-----------|----------|----------|
| **创业 Pitch** | 「做一个帮独居老人预约社区服务的微信小程序」 | 30 秒内出现完整思维导图 + 暖光动画 |
| **短视频选题** | 「职场新人如何向上管理」 | 9:16 竖屏动画，直接可发小红书 |
| **技术方案** | 「用 Event Sourcing 重构订单系统」 | 技术蓝图风格，架构节点逐步流转 |

每个 Demo 录制为 **15–30 秒屏幕录制 + 旁白**，在 Product Hunt、V2EX、即刻、小红书、B 站同步发布。

#### 2. 「Before / After」对比传播

```
Before：用户在 ChatGPT 里来回追问，复制粘贴到 Notion，再打开 XMind 手动建图……
After：一句话 → 微光 → 可分享动画。全程 30 秒。
```

这种对比图/短视频在社交媒体上的传播效率远高于功能介绍。

#### 3. 开源 + 故事化

- 在 GitHub README 首屏放 **GIF 动图**（思维导图生成过程 + 动画切换）
- 撰写「为什么我做灵感微光」创始人故事：解决自己 brainstorm 时的真实痛点
- 标注 Star 目标与技术栈标签（`ai` `mindmap` `react` `spring-boot` `llm`），吸引开发者 Star

---

### 阶段二：建立「使用习惯」（1 → 100 用户）

#### 4. 「每日一微光」挑战活动

- 官方每天发布一个开放题：「如果 AI 能帮你做一件事，你希望是什么？」
- 用户提交灵感 → 生成动画 → 带 `#灵感微光` 标签分享
- 每周评选「最佳灵感」，官方转发 + 定制 Skill 奖励

低成本 UGC，同时产出大量真实使用案例。

#### 5. 模板化降低首次使用门槛

在画布空白态提供 **一键示例**：

- 「产品 PRD 框架」
- 「竞品分析框架」
- 「个人 IP 定位框架」
- 「读书/note 结构框架」

用户点击示例 → 自动填入 prompt → 立即看到效果。减少「不知道该输入什么」的流失。

#### 6. 嵌入现有工作流

| 集成方向 | 价值 |
|----------|------|
| 浏览器插件 | 选中网页文字 → 一键发送到灵感微光 |
| 飞书 / 钉钉机器人 | 群内 brainstorm 实时出图 |
| Obsidian / Notion 插件 | 双向同步灵感与会话 |
| 微信 / Telegram Bot | 移动端随时记录灵感 |

每打通一个渠道，就获得该生态的自然流量。

---

### 阶段三：建立「增长飞轮」（100 → 10000 用户）

#### 7. 可分享的「灵感卡片」

- 每个会话生成一张精美封面图（核心节点 + 标题 + 二维码）
- 动画支持「分享链接」—— 无需安装，浏览器打开即可观看
- 卡片底部带「我也来微光一下 →」引流按钮

让每一次分享都成为获客入口。

#### 8. Skill 生态与开发者社区

- 开放 Skill 编写规范（已有 Markdown 基础）
- 举办 **Skill 创作大赛**：最佳动画风格收录官方
- 建立 Discord / 飞书社群，开发者交流 LLM Prompt、布局算法、新 Provider 接入

参考 Cursor Rules / Claude Skills 的增长路径——**工具 + 生态 = 护城河**。

#### 9. 垂直场景切入（避免大而全）

不要一开始就说「万能思维工具」。选一个垂直场景打透：

| 垂直场景 | Slogan 示例 |
|----------|---------------|
| 独立开发者 | 「把你的 Side Project 想法变成路演动画」 |
| 产品经理 | 「需求评审前，用 30 秒生成方案全景图」 |
| 短视频创作者 | 「选题 → 大纲 → 分镜动画，一条链路」 |

垂直场景的 SEO 和社群口碑远好于泛定位。

#### 10. 数据叙事：用案例证明 ROI

收集并发布用户故事：

> 「我用灵感微光准备创业比赛 pitch，原本需要 2 天做 PPT，现在 10 分钟搞定动画 demo，评委反馈『一目了然』。」

真实数字（节省时间、提升转化率）比任何功能描述都有说服力。

## 六、结语

灵感微光的价值不在于「又做了一个 AI 工具」，而在于捕捉了一个被忽视的瞬间——**想法刚刚诞生、尚未成形的那一秒**。

大多数工具假设用户已经知道要什么；灵感微光假设用户只有一个模糊的方向，并负责把它 **照亮、结构化、可表达**。

如果推广只强调技术栈，它只是一个 LLM 套壳；如果推广强调 **「让想法被看见」** 的情绪价值，它就有机会成为创意工作者离不开的第一站。

**微光虽小，足以引路。**
